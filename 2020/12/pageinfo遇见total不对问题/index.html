pageInfo遇见total不对问题

十二月 22, 2020 发布在 java, Essays

pageInfo相关注意事项

欸 朋友们好啊 我是中科软一名java小员工，目前正在北京总部出差 刚才有同事问我分页查询用的pageInfo啦？ 我说怎么回事 然后给我发了几张他postman测试接口截图 我一看 哦 原来是前两天 他分页查询用了个插件 一个是pageHelper 一个是插件的pageInfo类 当他今天测试接口的时候发现 返回的数据总条数和数据库总条数对应不上 我去看我的分页查询逻辑，发现不对劲 经过我仔细研究，我发现：

    //设置分页 pageNo-当前页数 pageSize-每页显示数据总数
    PageHelper.startPage(pageNo, pageSize);

    //进入数据库查询列表数据 pageHelper在这里做一次拦截器操作
    List<ListVo> listVos = renewalDetailsMapper.queryList(dto);

    //使用PageInfo包装返回数据结果
    PageInfo<ListVo> pageInfo = new PageInfo<>(listVos);

这三行语句必须写在一起 这样total才是总数 插件中间只能有一次查询语句，不能再做其他处理代码（两行代码之间不能再有其他代码）！

我原来写法是先查出列表数据，然后调用其他服务进行具体字段的值转化，最终再用pageInfo接收并返回：

    //设置分页 pageNo-当前页数 pageSize-每页显示数据总数
    PageHelper.startPage(pageNo, pageSize);

    //进入数据库查询列表数据 pageHelper在这里做一次拦截器操作
    List<Do> listVos = renewalDetailsMapper.queryList(dto);

    List<ListVo> vos=new ArrayList<>();
    //循环列表处理具体
    listVos.forEach(do -> {

        //具体处理代理。。。
        vos.add();

    });

    //使用PageInfo包装返回数据结果
    PageInfo<ListVo> pageInfo = new PageInfo<>(vos);

这样的写法，在pageInfo接收返回数据的代码中，自己新建了个返回vo的list。破坏了原有的total数， pageInfo.getTotal()一直是当前的 pageSize的总数，而不是查询的符合条件的数据库总条数。

注意：PageHelper只对紧跟着的第一个SQL语句起作用

发现了问题，下面就可以着手解决

方法1:

    //设置分页 pageNo-当前页数 pageSize-每页显示数据总数
    Page page = PageHelper.startPage(pageNo, pageSize);

    //进入数据库查询列表数据 pageHelper在这里做一次拦截器操作
    List<Do> listVos = renewalDetailsMapper.queryList(dto);

   List<ListVo> vos=new ArrayList<>();
    //循环列表处理具体
    listVos.forEach(do -> {

        //具体处理代理。。。
        vos.add();

    });

    //使用PageInfo包装返回数据结果
    PageInfo<ListVo> pageInfo = new PageInfo<>(vos);

    //手动设值  
    pageInfo.setPages(page.getPages());//总页数
    pageInfo.setTotal(page.getTotal());//总条数

PageHelper.startPage会返回一个page对象，这个对象在查询结果出来后会把页数，记录总数给page对象,用page.getPages()和getTotal()获取页数和记录总数.

方法2：

    //设置分页 pageNo-当前页数 pageSize-每页显示数据总数
    PageHelper.startPage(pageNo, pageSize);

    //进入数据库查询列表数据 pageHelper在这里做一次拦截器操作
    List<Do> listVos = renewalDetailsMapper.queryList(dto);

    //使用PageInfo包装返回数据结果
    PageInfo<Do> pageInfo = new PageInfo<>(vos);

啊 我大意了啊 没有闪 这个分页插件 不讲武德 来 骗 偷袭 我这个连续12天加班深夜的老同志 这好吗 这不好 欸 我劝各位年轻人 耗子尾汁 不要直接拿到pageHepler插件就直接用 一定要记住total这个坑 谢谢朋友们

springboot集成mybatis的pagehelper插件jar包
<!--引入PageHelper分页插件 → PageHelper-->
      <dependency>
          <groupId>com.github.pagehelper</groupId>
          <artifactId>pagehelper</artifactId>
          <version>5.1.2</version>
      </dependency>
      <dependency>
          <groupId>com.github.pagehelper</groupId>
          <artifactId>pagehelper-spring-boot-autoconfigure</artifactId>
          <version>1.2.5</version>
      </dependency>

      <dependency>
          <groupId>com.github.pagehelper</groupId>
          <artifactId>pagehelper-spring-boot-starter</artifactId>
          <version>1.2.5</version>
      </dependency>
注：这三个依赖引入后，可以不用再单独做插件的集成配置。使用pageHepler时尽量把上文三行代码写在一起，并且一定要有顺序！！！
